Mapper Library Example

Initial Database Design

This section describes the database structure and includes the SQL to create and populate the tables.

1 - Create User Accounts/Schemas

User Accounts
User Id Password Roles and Privileges Description
LIBRARIAN SHELVES Connect, Resource, Schema owner This is the owner of the schema. This account has full privileges to all objects in the schema.

2 - Create Table

LOCATION table
Column Data Type Size Constraints Description
LOCATION_ID NUMBER 10 Primary Key Unique identifier for the location
DESCRIPTION VARCHAR2 40 NOT NULL Description of the location

BOOK Table
Column Data Type Size Constraints Description
CATALOG_NO VARCHAR2 10 Primary Key Unique Identifier for item
TITLE VARCHAR2 50 NOT NULL Title for item
KEYWORDS VARCHAR2 30 Keywords for item for searching
DATE_ADDED DATE n/a NOT NULL Date item was added to inventory
COST NUMBER 6,2 NOT NULL Purchase cost of this item
LOCATION_ID NUMBER 10 Foreign Key Indicates where the book is located

3 - Create the Database Schema (Oracle)

First create the user account(schema),

SQL:

create user librarian identified by shelves default tablespace users temporary tablespace temp

Then grant that user the ability to connect and create tables

SQL:

grant resource, connect to librarian

4 - Create the Database Tables

First create the LOCATION table as outlined in the design section.

SQL:

CREATE TABLE LOCATION (
  LOCATION_ID number(10) constraint LOCATION_PK PRIMARY KEY,
  DESCRIPTION varchar2(40) NOT NULL
)


Now create the BOOK table as outlined in the design section.

SQL:

CREATE TABLE BOOK (
  CATALOG_NO varchar2(10) constraint BOOK_PK PRIMARY KEY,
  TITLE varchar2(50) NOT NULL,
  KEYWORDS varchar2(30),
  DATE_ADDED date NOT NULL,
  COST number NOT NULL,
  LOCATION_ID number(10)
)
